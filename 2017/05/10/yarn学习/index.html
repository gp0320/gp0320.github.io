yarn学习

2017-05-10

命令

当然可以加参数 –yes/-y 来自动回答所有的问题(yes),便捷的生成一个package.json

全局安装 yarn global add bagname

yarn add [–exact/-E]

用 –exact 或者 -E 会安装包的精确版本。 默认是安装包的主要版本里的最新版本。 比如说， yarn add foo@1.2.3 会接受 1.9.1 版，但是 yarn add foo@1.2.3 –exact 只会接受 1.2.3 版。

yarn add [–tilde/-T]

用 –tilde 或者 -T 来安装包的次要版本里的最新版。 默认是安装包的主要版本里的最新版本。 比如说，yarn add foo@1.2.3 –tilde 会接受 1.2.9，但不接受 1.3.0。

删除依赖

命令yarn remove [package] 删除某个依赖，并自动更新package.json和yarn.lock文件

更新依赖

命令yarn upgrade [package]@[version/tag]
更新某个依赖的版本，并自动更新package.json和yarn.lock文件

特色

1、离线工作

  1. 从多个注册表安装
    从远程 gzip 压缩文件安装包
    yarn add https://thatproject.code/package.tgz

从你的本地文件系统安装包，如下:
yarn add file:/path/to/local/folder

从远程 git 仓库安装包，如下:
yarn add

  1. 快速获取安装包
    在全局缓存目录查看是否下载过这些包。如果没有，Yarn 会获取原始包，并将其放入全局缓存，以保证可以离线工作和无需重复安装。4. 4、自动锁定安装包版本、
    yarn.lock 锁定了安装包的精确版本以及所有依赖项。有了这个文件，你可以确定项目团队的每个成员都安装了精确的软件包版本，部署可以轻松地重现，且没有意外的 bug。

6、扁平模式
对于多个包依赖同一个子包的情况，yarn会尽量提取为同一个包，防止出现多处副本，浪费空间。比如1.2中，yarn会为babel-generator和babel-helper-define-map 创建同一个lodash子依赖，这样就节约一份的空间。
